学习渗透的基础所做的笔记（前端，后端）

通信原理

linux体积小，开源, 没有盘符/home/
centos debian ubuntu
andriod基于linux
windows家用办公, 有盘符C:/Users/
mac os也没有盘符
服务器 简单来说是24小时有公网ip的电脑(服务器指的是网络环境下为客户机提供某种服务的专用计算机)
服务器属于计算机的一种，高速cpu运算能力，长时间运行，强大的I/O，更好的扩展性

ipconfig(cmd命令)查内网ip地址
公网地址由运营商分配
内网地址由路由器分配(10.0.0.0~10.255.255.255)
DNS协议 将域名翻译为ip
域名由顶级域名(收费).二级域名.三级域名
端口是网络接口(范围2^16-1~~0)netstat -an(cmd)查端口
端口作为客户端和服务端的间接层，起到类似窗口的功能

http协议 超文本传输协议
https基于http协议和ssl构建的可加密传输和身份认证的协议
GET POST
GET能直接看到传参数据  标识符：？有长度限制
POST不能看到传参数据 长度无限制
user-agent传参是告诉浏览器是什么，系统是什么

浏览过程
1输入域名或ip
2浏览器向DNS服务器查询ip
3浏览器向查询到的ip的web服务器发送http请求
4服务器接受请求，返回http响应
5浏览器提取状态，协议编码，正文的有效性息，然后翻译成图形界面显示

web容器(IIS（windows）,Apache,Nginx等) 可以向发出请求的浏览器提供html等文档或者图像，脚本等的程序

网站

由web容器，数据库，动态语言，
动态语言是对服务器的行为的编程，称为服务器脚本(php,jsp,python,asp)
动态的向web编辑，对html表单提交的用户进行访问，定制网页，提高安全性
集成搭建环境，jsp(tomcat),php(phpstudy,lnmp)
访问网站就是访问网站服务器的文件

html超文本标记语言，

SQL

(结构化查询与语言)
将大量数据保存，通过计算机加工的可以高效访问的数据集合
oracle sqlserver db2 postgresql mysql access
服务端：接受处理请求
客户端：向服务端发起请求
库：一堆表组成的数据集合
表的列是字段，行是记录，
sql语句：用表名列名组合的一条语句
DDL：创建，删除，修改，数据库和表等对象
DML：查询或修改表中的记录
DCL：确认或取消数据库中的数据变更的执行操作，对象权限的设定
use 数据库;
set names utf8 设置使用的字符集;
selectfrom 表名;
select updata delete 提取 更新 删除 数据库的数据
insert into 向数据库插入
create database 创建新数据库
alter database 修改数据库
create table 创建新表
alter table 改变表
drop table 删表
create index 创建索引
drop index 删除索引
select 向数据库中选取数据并储存在结果表中
select 字段名(可以选取多个，如果不选取则默认全部字段) from 表名
select distinct（不同）返回唯一不同的值
selectfrom 表名 where 条件 ；过滤记录
sql用单引号引用文本（大部分也接受双引号）
between 在某个范围内
in 列中可能值
order by 字段名 asc(升序)|desc(降序) 对结果集按照一列或几列进行排序
insert into 表名 (列名) values 值（如果不写列名则需要写出每一列数据）
updata 表名 set 字段=值 where 条件
delete from 表名 where 条件
%代替一个或多个字符
模糊查询 selectfrom (表) where like (字段)==%(字符)
联合查询 union 将两个结果放在同一个集合表中
子查询 selectfrom user where username =(select username from admin where id=1);
group_concat(col)返回一组数据
ascii(char)
database()数据库名
user() or system_user()返回用户名
version()
sleep()
